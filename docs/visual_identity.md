# Visual Identity Specification

This document defines the core aesthetic philosophy and emotional anchors of Tracker OS. Its purpose is to preserve the look, feel, and mood of the application as new features are added over the coming years.

---

## 1. How Tracker OS Should Feel

* **Calm**: Low cognitive load, high contrast but low color saturation, clean boundaries, ample white space.
* **Professional**: Built for daily usage over years, utilizing standard system typography, sharp edges, and zero decorative elements.
* **Dense but Readable**: Compact spacing and list heights optimized for desktop productivity, utilizing variable font weights rather than large sizes.
* **Minimal**: Every pixel must serve a clear purpose. No motivational text, no decorative lines, and no decorative widgets.

---

## 2. Visual Style Guidelines (Anti-Goals)

To prevent visual drift, the following design concepts are **strictly prohibited** in Tracker OS:

* **No Gradients**: All backgrounds must be solid. Accent borders and text must use solid tokens.
* **No Glassmorphism**: No semi-transparent backing cards or blurry background panels.
* **No Neumorphism**: No dual-shadow embossed buttons or bubble elements.
* **No Neobrutalism**: No hard black thick borders, off-kilter shadows, or primary saturated background fills.
* **No Floating Cards**: Cards must be anchored by a 1px border. No large soft shadow offsets.
* **No Flashy Animations**: Transitions must be standard fade or slide-in under 200ms duration. No bouncy, springy, or rotating loaders.

---

## 3. Aesthetic Influences

### Inspired By (The Core Philosophy)
* **Linear**: Information density, high-contrast dark modes, strict focus-visible keyboard navigation, minimal color accents.
* **Things 3**: Clean checkbox design, subtle task completion transitions, content-first simplicity.
* **Raycast**: Command-bar centric operations, keyboard efficiency, clear action lists.
* **Apple Calendar / Reminders**: Clear vertical agenda timeline, standard category indicators, collapsible advanced menus.
* **Apple Notes / GitHub Desktop**: Flat layout hierarchy, sidebars for listing, main text areas for editing, standard system sans-serif fonts.
* **Arc Browser**: Calm utility frames, flat sidebars, visual workspace segmentation.

### NOT Inspired By (Forbidden Styles)
* **Crypto Dashboards**: Pulsing indicators, flashy line graphs, dashboard widgets, and speculative telemetry widgets.
* **Gaming UIs**: Oversaturated color accents, angled layouts, glowing active states, and custom heavy fonts.
* **Dribbble Concepts**: Non-functional layouts, card blocks with arbitrary margins, and zero-state illustrations.
* **Material You**: Large bubbly buttons, heavy spacing gaps, and rounded circular bounds.
* **Enterprise Admin Templates**: Redundant stats cards, complex multi-color bar charts, and nested header tabs.
